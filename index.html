Dev / Security Docs

Security Architecture

Implementation reference for engineering teams. Hide this section in production with a feature flag.

🔐

Authorization

JWT stored in httpOnly, Secure, SameSite=Strict cookie. Patients can only read/write their own appointment rows. Staff scoped to their branch_id.

SELECT * FROM appointments
WHERE patient_id = :user_id
AND branch_id = :user_branch;
🧪

Input Validation

All form inputs validated client-side (pattern, type, maxlength) AND server-side. HTML stripped with DOMPurify. Parameterized queries only — no string concatenation in SQL.

// Phone: /^09\d{9}$/
// Email: RFC 5321 regex
// Name: /^[A-Za-zÀ-ÿ\s\-]+$/
🌐

CORS Policy

Server allows only cosmodentalph.com origin. Preflight OPTIONS handled. Credentials flag enabled only for same-origin requests.

Access-Control-Allow-Origin:
https://cosmodentalph.com
Access-Control-Allow-Credentials: true
🚦

Rate Limiting

Express-rate-limit: /api/book → 5 req/min/IP. /api/auth → 10 req/min/IP. Returns 429 + Retry-After header. Frontend shows friendly message (not raw 429).

// Cost protection: email/SMS cap
// Hard limit: 1000 OTPs/day
// Alert at 80% threshold
🔑

Password Reset Security

Tokens are cryptographically random (32 bytes). Expire in 15 minutes. Single-use — invalidated on click. No user enumeration (same response for existing/non-existing emails).

crypto.randomBytes(32).toString('hex')
expires_at: NOW() + INTERVAL 15 min
used: false → true on first use
⚠️

Error Handling

Frontend catch blocks show friendly messages only. Stack traces and DB errors never reach the client. All errors logged server-side with request_id for tracing.

catch(err) {
log.error({err, request_id})
res.status(500).json({
message: "Something went wrong"
})
}
🗄️

Database Indexes

Index all frequently queried fields to keep appointment lookups fast as data grows.

CREATE INDEX idx_appt_patient
ON appointments(patient_id);
CREATE INDEX idx_appt_date
ON appointments(appt_date);
CREATE INDEX idx_user_email
ON users(email);
📋

Logging & Alerts

Structured JSON logs to stdout → CloudWatch / Datadog. PagerDuty alert on: 5xx rate >1%, booking failures >5/min, auth failures >20/min.

log.info({ event: 'appt_booked',
patient_id, branch, service,
timestamp: new Date().toISOString() })
🔄

Rollback Strategy

Blue-green deployment on Railway / Render. DNS switch takes <2 min. DB migrations are always backward-compatible. Rollback runbook in Notion, reviewed quarterly.

// Feature flags via env vars
// BOOKING_ENABLED=true/false
// Revert: git revert + push to prod
